Is any data being collected?

{{mask}}{{ Nope! No data is logged or collected by any of our web proxies featured. Logging is disabled to ensure user privacy on the backend. NextDNS queries are adjusted to not log client IPs or domains and store in Switzerland. Cloudflare is utilized for this project to improve performance and have easier DNS management due to the scope of this project. At a minimum Cloudflare numerical values are utilized for tracking visits and page views indicating when server upgrades are needed. NGINX is set to store no logs. Security

{{mask}}{{ InvisiProxy LTS is built from the ground up to ensure a secure and reliable web proxy service for its users. One of the core technologies behind InvisiProxy LTS is the new Wisp protocol, developed by Mercury Workshop.

This protocol brings several innovative features and enhancements that contribute to the speed and security of the service over previous models. }}

{{mask}}{{ The Wisp protocol is designed to handle modern web traffic demands while maintaining high security standards. By implementing Wisp, this project leverages the following benefits: }}

{{mask}}{{ Performance: The Wisp protocol optimizes TCP data transfer speeds, ensuring that users experience minimal latency while browsing. }}

{{mask}}{{ Compatibility: Wisp is built to be compatible with a wide range of web technologies, ensuring seamless access to various websites and online services. This is done through the support of various open-source transports following the protocol (Epoxy Transport and Libcurl Transport). }}

{{mask}}{{ Resilience Against Censorship: The protocol is designed to bypass sophisticated censorship techniques employed by restrictive networks, allowing users to access blocked content with ease. }}

{{mask}}{{ Fingerprint Resistance: This project has strong fingerprint resistance mechanisms within its reverse proxy setup for official instances. This means that the proxy is designed to minimize the unique identifiers that can be used to track or identify users, thereby enhancing privacy. By following industry best practices and continually updating its methods, users can browse anonymously without leaving identifiable traces. }}

{{mask}}{{ Transparency: Transparency is a key principle for InvisiProxy LTS. The project operates with an open-source model, allowing the community to review and contribute to the codebase. This openness not only fosters trust but also enables continuous improvement through community feedback and collaboration. By adhering to transparent practices, InvisiProxy LTS ensures that users are aware of how their data is handled and what measures are in place to protect their privacy. }}

{{mask}}{{ Source/Global Randomisation: In order to further enhance security and privacy, InvisiProxy LTS employs source and global randomisation. This allows the project to remain unblocked longer on domains and also improve anonmity while browsing. Typical web proxies will leave traces whenever rewriting assets, but with this method, it is much harder to detect the proxy and block it. }}

{{mask}}{{ Encryption: InvisiProxy LTS utilizes advanced proxy technologies that focus on end-to-end encryption and secure data transfer, particularly in the context of URL rewriting proxies. NOTE: This is still a massive work in progress and open to contributions. }}

{{mask}}{{ Regular Audits: Conducting regular security audits to identify and address potential vulnerabilities. }}
